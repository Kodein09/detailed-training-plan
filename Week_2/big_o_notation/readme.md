# Big Order Notation Или Большое О

### Это способ определения или описания эффективности какого либо алгоритма

* Временная сложность показывает сколько уйдёт затраченного времени на выполнения алгоритма (показывает как быстро выполнится алгоритм, зависит от входных данных)
* Пространственная сложность показывает сколько ресурсов памяти используется
Это важно по причине необходимости масштабировать алгоритмы

### Основные обозначения О большого:
| Обозначение | Название                | Пример                                                                   |
|-------------|-------------------------|--------------------------------------------------------------------------|
| О(1)        | Константная             | Доступ к элементу списка по индексу                                      |
| О(log n)    | Логарифмическая         | Бинарный поиск, начиная с центра массива делая шаги деля каждый шаг на 2 |
| О(n)        | Линейная                | Линейный поиск, проходимся по всему списку по каждому элементу           |
| O(n log n)  | Линейно-логарифмическая | Сортировка слиянием, быстрая сортировка                                  |
| O(n^2)      | Квадратичная            | Вложенные циклы (сравнение всех со всеми)                                |
| O(2^n)      | Экспоненциальная        | Перебор всех подмножеств                                                 |
| O(n!)       | Факториальная           | Полный перебор перестановок                                              |

### Начнём с временной сложности
Что такое "время выполнения" и как его считать</br>
* Анализ простых циклов (O(1), O(n), O(n²))</br>
* Вложенные циклы и зависимости</br>
* Логарифмическая сложность</br>
* Рекурсия и как анализировать её (O(2ⁿ) и O(log n))</br>
* Примеры с сортировкой</br>
* Амортизированная сложность</br>
* Пространственная сложность (сравнение с временной)</br>

### Теория: Что такое время выполнения
Количество операций, которые выполняет алгоритм в зависимости от размера входных данных `n`. Это не про секунды, а про рост числа операций при увеличении `n`.</br>
#### Константная:
    def print_hello():
        print("Hello")

* Эта функция всегда выполняется за одно и то же время, независимо от `n`
* Сложность данного алгоритма: O(1) - константная.

#### Линейная:
      def print_item(lst):
          for item in lst:
              print(item)

* Количество итераций зависит от размера `lst`.
* Если `lst` длиной в 1000, значит и `print` выполнится 1000 раз.
* Сложность данного алгоритма: O(n) - линейная, где `n = len(lst)`.

#### Счёт операций
*Пример кода (анализ времени выполнения)*

    def sample_function(arr):
        total = 0               # O(1)
        for i in arr:           # O(n)
            total += 1          # O(1) * n = O(n)
        return total            # O(1)
Общая сложность: O(n)

#### Разберём пример одного линейного алгоритма

    def process_array(arr):
        for i in arr:       # O(n)
            print(i)        # O(n)
        print("Done")       # O(1)

Цикл `for` в данном примере имеет сложность `O(n)`, поскольку `i` проходит по всем элементам массива `arr` - `for i in range`.</br>
Вторая строка `print(i)` выводит один из элементов массива, внутри цикла будет иметь константную сложность `O(1)`, но, поскольку неизвестно, сколько раз будет происходить вызов - имеет сложность `O(n)`.</br>
Третья же строка `print("Done")` вызывается лишь раз, поэтому имеет константную сложность `O(1)`.</br>
Исходя из наших выводом можем давать общую сложность этому алгоритму: `O(n) + O(1) = O(n)`, итог наш алгоритм `линейный`.


### Теория: Вложенные циклы
Когда один цикл находится внутри другого цикла, выполняется по всем сочетаниям:

    for i in range(n):
        for j in range(n):
            ...
* Внешний цикл: `n` раз
* Внутренний цикл: каждый раз по `n`
* Итого: `n * n = n^2`
* Сложность: `O(n^2)`

### Теория: Несимметричные циклы (O(n * m))

    for i in range(n):
        for j in range(m):
            ...
* Внешний цикл: `n` раз
* Внутренний цикл: каждый раз по `m`
* Итого: `O(n * m)`
* Если `m == n` -> `O(n^2)`
* Если `m` равняется 10, `m = 10`, тогда `O(n * 10)` -> упрощается до `O(n)`

### Теория: Внутренние зависимости

    for i in range(n):
        for j in range(i):
            ...
* При `i = 0` -> 0 раз
* `i = 1` -> 1 раз
* `i = 2` -> 2 раза
* ...
* `i = n-1` -> n-1 раз
* Суммарно: `0 + 1 + 2 ... + (n-1)` = `n(n-1)/2` -> `O(n^2)`
